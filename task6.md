Explain why staying in sync matters (avoids conflicts, prevents lost work, supports collaboration). (AC6.2)

Staying in sync with the remote repository and your teammates matters because:

- It avoids conflicts: frequent pulls and merges keep changes small and easier to reconcile, reducing painful merge conflicts.
- It prevents lost work: regularly pushing and pulling ensures work is backed up and visible to others, so changes aren't accidentally overwritten or discarded.
- It supports collaboration: a shared, up-to-date codebase enables effective code review, coordinated development, and faster feedback loops.

Keeping branches current and communicating changes helps the team move faster with less friction.
